# Mathin Terms「知识星系」设计方案

> 本文档用于替代此前将 `/terms` 做成全量知识图谱的方向。  
> 新方向：**Terms 是一个小王子式的 3D 数学知识星系**。  
> 星球负责领域入口，岛屿负责主题入口，具体知识点只在岛屿学习路径与概念页中出现。
> 星球和岛屿最终实现效果参考游戏围攻的低多边形菜单，展示图在同目录文件夹中

---

## 0. 核心结论

Mathin 的 Terms 板块不应直接呈现一张完整知识图谱，而应采用四层结构：

```txt
星系看领域
星球看主题
岛屿看路径
节点学概念
图谱看关系
```

对应页面结构：

```txt
/terms
  数学知识星系首页：展示若干颗知识星球

/terms/[planet]
  单颗星球聚焦页：展示该星球上的多个知识岛屿

/terms/[planet]/[island]
  岛屿学习路径页：展示具体知识点路径

/terms/concepts/[slug]
  概念详情页：展示定义、例子、图示、前后置关系

/terms/graph
  完整知识图谱高级视图：给教师、编辑者、重度用户使用
```

重要修正：

1. **第一阶段就使用 3D**，不再用纯 SVG / 图片模拟星球。
2. 初期可以让所有星球使用同一个低多边形基础模型，只通过颜色、缩放、位置、标签、少量地标进行区分。
3. 每颗星球最多岛屿数量不要太多，由最终的知识点聚合来划分。
4. `/terms/[planet]` 与 `/terms/[slug]` 会发生路由冲突，因此不采用这两个并列动态路由。
5. 星球无论放大到什么状态，都必须首先像“球”，不能变成俯视大陆。

## 1. 产品定位

### 1.1 Terms 的定位

Terms 是 Mathin 的核心板块。它不是普通文章列表，也不是单纯的知识图谱，而是一个可探索的数学学习宇宙。

它需要同时承担三件事：

| 目标 | 说明 |
|---|---|
| 学习入口 | 帮用户知道从哪里开始学 |
| 知识组织 | 把小学到大学基础数学知识组织成可理解的结构 |
| 视觉世界观 | 承接 Mathin 的《小王子》式星球宇宙气质 |

### 1.2 核心隐喻

| 视觉对象 | 产品含义 |
|---|---|
| 星系 | 整个数学知识世界 |
| 星球 | 一个数学大领域 |
| 岛屿 | 一个领域内的主题模块 |
| 路径 | 该主题内的学习顺序 |
| 节点 | 一个具体知识点 |
| 星光连线 | 前置、后续、相关关系 |
| 完整图谱 | 高级关系视图，不作为首页 |

## 2. 页面层级与路由设计


```txt
/terms
/terms/[planet]
/terms/[planet]/[island]
/terms/concepts/[slug]
/terms/graph
```

| 路由 | 页面 | 用途 |
|---|---|---|
| `/terms` | 知识星系首页 | 展示全部知识星球 |
| `/terms/[planet]` | 星球聚焦页 | 展示某颗星球上的 3 个岛屿 |
| `/terms/[planet]/[island]` | 岛屿学习路径页 | 展示该岛屿内的知识点路径 |
| `/terms/concepts/[slug]` | 概念详情页 | 展示单个概念内容 |
| `/terms/graph` | 完整知识图谱 | 高级关系视图 |

## 3. 视觉总原则

### 3.1 星球必须像球

这是整个设计最重要的判断标准。

正确方向：

```txt
小小的圆形星球
表面有少量凸起的低多边形岛屿
可以看到星球弧面
岛屿像贴在球面上的大地标
整体缓慢自转
有轻微大气光晕
```

视觉效果靠近围攻游戏菜单

错误方向：

```txt
一整块俯视大陆
一大堆岛屿平台铺满画面
像 RTS 地图
像普通关卡选择地图
没有球体轮廓
文字和节点挤在模型表面
```

### 3.2 首页只显示星球，不显示知识点

`/terms` 首页只回答一个问题：

```txt
数学世界有哪些大区域可以探索？
```

首页不展示：

```txt
具体知识点名
知识点节点
密集连线
全量图谱
大量岛屿名
```

首页最多展示：

```txt
7 颗星球
每颗星球 1 个名称
每颗星球 1 行短说明
当前推荐星球的微弱高亮
```

### 3.3 具体知识点只在岛屿页出现

层级约束：

| 页面 | 可以显示 | 禁止显示 |
|---|---|---|
| `/terms` | 星球 | 知识点节点、全量关系线 |
| `/terms/planets/[planet]` | 3 个岛屿 | 岛屿内全部知识点 |
| `/terms/planets/[planet]/islands/[island]` | 8–12 个路径节点 | 其他星球的大量内容 |
| `/terms/concepts/[slug]` | 当前概念与一阶关系 | 全量图谱 |
| `/terms/graph` | 完整图谱 | 作为默认首页 |

## 4. 3D 技术方向

### 4.1 第一阶段必须使用 3D

本设计将 3D 作为第一阶段的刚性要求。原因：

1. 低多边形小星球是核心视觉，不是装饰。
2. 仅用图片或 SVG 很难表达“星球旋转”和“点击推进”的空间感。
3. 用户明确希望参考《围攻》菜单中的小星球选择体验。

---

### 4.2 第一阶段模型策略

初期不要一次性制作 7 套复杂星球模型。

采用：

```txt
一个通用低多边形星球模型
复用到全部 7 颗星球
通过颜色、位置、缩放、光晕、标签区分
```

第一阶段星球可以保持一致：

| 元素 | 第一阶段策略 |
|---|---|
| 星球主体 | 同一个 low-poly sphere / ico sphere |
| 岛屿 | 同一套 多 个岛屿位置模板 |
| 地标 | 可先用简单几何体占位 |
| 材质 | 每颗星球换主色即可 |
| 动效 | 全部使用同样的慢速自转 |
| 交互 | hover、click、focus 先打通 |

---

### 4.4 低多边形模型规范

#### 星球主体

星球主体是一个完整的球形

#### 岛屿

岛屿应是星球表面的凸起，而不是平铺大陆。

建议做法：

```txt
在球面上放 相应数量的 低多边形凸起平台
平台之间留出明显球面空隙
```

禁止：

```txt
岛屿铺满整个正面
多个平台连成一整块大陆
岛屿边缘超出球体轮廓太多
```

#### 地标

第一阶段可用简单几何体：

```txt
box
cone
cylinder
torus
simple extruded shape
```

不要一开始追求复杂模型。

---

### 4.5 性能约束

首版目标：

```txt
7 颗星球同时出现在首页时，桌面端流畅，移动端可降级。
```

建议限制：

| 项目 | 建议 |
|---|---:|
| 单颗星球三角面 | 尽量 < 5k |
| 首页总三角面 | 尽量 < 40k |
| 动画速度 | 慢速，避免持续高频复杂计算 |
| 阴影 | 首版少用或不用实时阴影 |
| 后处理 | 首版不加重型 bloom / DOF |
| 移动端 | 可降低星球数量、关闭部分粒子 |

必须支持：

```css
prefers-reduced-motion
```

当用户开启减少动态效果时：

```txt
停止星球自转
禁用大幅镜头推进
保留静态 hover / focus 效果
```

## 5. 7 颗知识星球

建议设置 7 颗知识星球。

5 颗太少，会把知识硬塞在一起；10 颗太多，首页会散。7 颗可以覆盖小学到大学基础数学的主要结构。

| 编号 | 星球 | 英文 | 定位 |
|---:|---|---|---|
| 1 | 数与数量 | Number & Quantity | 数的产生、表示、大小、比例、量感 |
| 2 | 运算与算法 | Operations & Algorithms | 加减乘除、运算律、算法、估算 |
| 3 | 图形与空间 | Geometry & Space | 图形、测量、空间、变换 |
| 4 | 关系与代数 | Algebra & Relations | 变量、等式、方程、结构 |
| 5 | 函数与变化 | Functions & Change | 坐标、函数、变化率、极限与微积分 |
| 6 | 数据与概率 | Data & Chance | 统计、概率、数据理解 |
| 7 | 逻辑与证明 | Logic & Proof | 命题、集合、证明、数学结构意识 |

---

## 6. 星球视觉设定

第一阶段所有星球可以共用同一模型，只做颜色与地标差异。

| 星球 | 主色 | 气质 | 可见地标方向 |
|---|---|---|---|
| 数与数量 | 草绿 / 暖金 | 起点、生命感、基础 | 数字石块、计数塔、小树 |
| 运算与算法 | 冰蓝 / 钢灰 | 工坊、机械、秩序 | 齿轮、桥梁、算盘结构 |
| 图形与空间 | 紫蓝 / 月白 | 晶体、几何、空间感 | 棱锥、水晶、圆规塔 |
| 关系与代数 | 青绿 / 深墨 | 符号、关系、结构 | x/y 符号石碑、等号桥 |
| 函数与变化 | 橙黄 / 沙金 | 时间、运动、变化 | 沙漏、斜坡、钟塔 |
| 数据与概率 | 湖蓝 / 银灰 | 观测、统计、随机 | 图表碑、骰子、望远镜 |
| 逻辑与证明 | 深蓝 / 金白 | 灯塔、秩序、推理 | 灯塔、书库、门廊 |

统一高亮色：

```txt
温暖金色星光
```

不要每颗星球使用完全不同的高亮体系，否则星系会失去整体感。

## 7. 21 个知识岛屿设计

每颗星球先考虑 3 个岛屿。岛屿是主题入口，不是具体知识点，可以根据节点情况增加岛屿数量

---

### 7.1 数与数量星球

定位：建立“数是什么”“数量如何表示”“数量之间如何比较与转换”。

视觉：草地星球、暖金光、一棵小树、数字石块、小羊 / 玫瑰点缀。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 数的诞生岛 | 数字石块、计数木桩、1/2/3 小路 | 数数、自然数、0、整数、数轴初步 |
| 数的表示岛 | 位值塔、十进制台阶、数字牌 | 十进制、位值、大数、小数、百分数 |
| 分数比例岛 | 分数圆盘、比例桥、刻度尺 | 分数、小数互化、比、比例、百分率 |

---

### 7.2 运算与算法星球

定位：从数量变化进入可操作规则，包括加减乘除、运算律、算法意识。

视觉：冰蓝与钢灰，小型机械星球，齿轮、桥梁、算盘、传送带。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 加减工坊岛 | 加号桥、减号升降台、进位齿轮 | 加法、减法、进位、退位、估算 |
| 乘除磨坊岛 | 阵列农田、倍数风车、分组磨坊 | 乘法、除法、倍、余数、因数倍数 |
| 运算规则岛 | 规则石碑、括号门、分配律机械臂 | 运算顺序、括号、运算律、简便计算 |

---

### 7.3 图形与空间星球

定位：图形、测量、空间想象、变换与几何结构。

视觉：紫蓝色星球，水晶、棱锥、圆规塔，月光感。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 形的花园岛 | 点线面花园、三角形石门、圆形广场 | 点线面、角、三角形、多边形、圆 |
| 测量山谷岛 | 尺子桥、面积田、体积矿洞 | 长度、周长、面积、体积、单位换算 |
| 空间变换岛 | 镜面对称湖、旋转塔、坐标网格 | 位置、方向、平移、旋转、对称、坐标 |

---

### 7.4 关系与代数星球

定位：从具体数进入抽象关系，包括符号、变量、等式、方程、结构。

视觉：青绿色星球，符号石碑、等号桥、变量门。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 规律村岛 | 重复图案村落、数列台阶、图案旗帜 | 找规律、数列、符号、变量、代数式 |
| 方程门岛 | 天平、等号门、未知数钥匙 | 等式、未知数、方程、解方程、不等式 |
| 结构城岛 | 多项式城墙、因式分解工坊、矩阵门廊 | 多项式、因式分解、关系、映射、矩阵初步 |

---

### 7.5 函数与变化星球

定位：变化、对应、图像、函数、极限、微积分。

视觉：橙黄色星球，沙漏、钟塔、斜坡、流动光线。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 坐标港岛 | 坐标码头、网格地面、斜率桥 | 坐标、图像、正比例、一次函数、斜率 |
| 变化钟塔岛 | 钟塔、坡道、运动轨迹 | 变量关系、变化量、变化率、二次函数、指数变化 |
| 微积分悬崖岛 | 极限悬崖、切线灯塔、面积湖 | 极限、连续、导数、切线、积分 |

---

### 7.6 数据与概率星球

定位：数据收集、统计图表、概率、随机性、推断。

视觉：湖蓝色星球，观测台、图表碑、骰子石、望远镜。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 数据集市岛 | 表格摊位、柱状图牌坊、分类箱 | 分类、数据收集、表格、统计图、数据描述 |
| 统计灯塔岛 | 平均数灯塔、中位数石桥、离散云 | 平均数、中位数、众数、方差直觉、分布 |
| 概率花园岛 | 随机风车、样本空间花园、概率池 | 随机事件、样本空间、概率、独立事件、期望 |

---

### 7.7 逻辑与证明星球

定位：数学语言、逻辑、集合、证明方式、抽象结构意识。

视觉：深蓝色星球，金色灯塔、书库、石门、星光桥。

| 岛屿 | 视觉地标 | 主题 |
|---|---|---|
| 逻辑灯塔岛 | 灯塔、真值石碑、连接词桥 | 命题、真值、且或非、充分必要、量词 |
| 证明桥岛 | 石桥、反例洞穴、归纳阶梯 | 举例、反例、直接证明、分类讨论、反证法、归纳法 |
| 集合书库岛 | 书库、集合圆环、映射门 | 集合、元素、子集、交并补、关系、映射、无限集合 |

## 8. 首页 `/terms` 设计

### 8.1 页面目标

让用户感到：

```txt
Mathin 的数学知识不是一本书，
而是一片可以旅行的星系。
```

### 8.2 页面结构

```txt

  7 颗 3D 小星球组成星系
  星球在不同深度层缓慢漂浮
  当前推荐星球微微发光

```

### 8.3 星球排列

不使用整齐网格，采用不规则星系排列。

```txt
                 图形与空间

       运算与算法           函数与变化

 数与数量          中心轨道           关系与代数

       数据与概率           逻辑与证明
```

轨道线要极淡，只表达空间感，不表达强图谱关系。

### 8.4 首页交互

#### 默认状态

```txt
所有星球缓慢自转
星球大小不完全相同
远处星球更小、更暗
当前推荐星球轻微发光
```

#### Hover 星球

```txt
星球略微放大：1.04–1.08
显示星球名称
显示一句副标题
显示极少量地标轮廓
不显示知识点
```

#### 点击星球

```txt
镜头推进
其他星球退到背景
被点击星球放大到屏幕中央
进入 /terms/planets/[planet]
```

## 9. 星球聚焦页 `/terms/[planet]`

### 9.1 页面目标

展示某颗星球上的 3 个知识岛屿。

例如「数与数量星球」：

```txt
数与数量星球
  ├─ 数的诞生岛
  ├─ 数的表示岛
  └─ 分数比例岛
```

### 9.2 视觉约束

```txt
星球直径：屏幕高度的 60%–75%
球体轮廓完整可见
正面最多出现 3 个岛屿
每个岛屿有一个大型地标
岛屿名用外部 UI 标签标注，不直接写在模型上
```

禁止：

```txt
星球变成一整块大陆
所有岛屿摊开成地图
星球上写很多文字
学习路径画满星球表面
```

### 9.3 交互

#### 默认

```txt
星球居中
当前推荐岛屿发光
左侧显示星球介绍
右侧显示学习进度
```

#### Hover 岛屿

```txt
岛屿微微升起 / 发光
显示岛屿名
显示一句说明
显示“进入学习路径”
```

#### 点击岛屿

```txt
镜头靠近岛屿
进入 /terms/planets/[planet]/islands/[island]
```

## 10. 岛屿学习路径页 `/terms/[planet]/[island]`

### 10.1 页面目标

这里才展示具体知识点。

视觉参考多邻国路径，但不要做成普通平面列表。它应该像进入了一座小岛内部。

```txt
起点
  ↓
知识点 1
  ↓
知识点 2
  ↓
探索 / 练习
  ↓
知识点 3
  ↓
小结
```

### 10.2 视觉结构

```txt
一座低多边形岛屿
从码头 / 山脚开始
沿着小路走向塔、树、桥、灯塔
每个知识点是路径上的星石 / 小站
```

### 10.3 路径节点状态

| 状态 | 视觉 |
|---|---|
| 未开始 | 暗色星石 |
| 可学习 | 微亮星石 |
| 当前推荐 | 金色光圈 |
| 已完成 | 点亮 |

## 11. 概念详情页 `/terms/concepts/[slug]`

### 11.1 页面目标

展示一个具体概念的学习内容。

页面内容包括：

```txt
定义
直观解释
例子
反例
图示 / 交互组件
前置概念
后续概念
相关概念
```

### 11.2 局部星座图

概念页可以有一个小型局部关系图，但只显示当前概念的一阶关系。

显示：

```txt
当前概念
前置概念 2–4 个
后续概念 2–4 个
相关概念 2–6 个
```

禁止显示完整全局图谱。

## 12. 完整知识图谱 `/terms/graph`

### 12.1 定位

完整知识图谱是高级视图，不是学生入口首页。

适用对象：

```txt
教师
内容编辑者
重度学习者
想看完整结构的用户
```

### 12.2 功能

```txt
搜索概念
按星球筛选
按岛屿筛选
按学段筛选
显示 / 隐藏关系类型
点击节点打开概念详情
局部展开一阶 / 二阶关系
```

### 12.3 默认状态

默认不要全量显示所有标签。

```txt
只显示大节点
隐藏小标签
用户搜索、放大或点击后再显示标签
```

## 17. 给执行 agent 的最终说明


## 视觉要求

1. 星球必须首先像“球”。
2. 选中星球放大后，也必须保留完整球体轮廓。
3. 每颗星球最多 3 个主要岛屿。
4. 首页只展示星球，不展示具体知识点。
5. 岛屿只作为主题入口，具体知识点只在岛屿路径页展示。
6. 不要出现类似 Obsidian 全局图谱的密集节点线。
7. 完整知识图谱只放在 `/terms/graph`。

## 七颗星球

1. 数与数量 Number & Quantity
2. 运算与算法 Operations & Algorithms
3. 图形与空间 Geometry & Space
4. 关系与代数 Algebra & Relations
5. 函数与变化 Functions & Change
6. 数据与概率 Data & Chance
7. 逻辑与证明 Logic & Proof

## 第一阶段目标

- 打通 3D 星系首页。
- 打通星球点击进入聚焦页。
- 打通岛屿点击进入学习路径页。
- 确认 `/terms/concepts/[slug]` 概念页路由。
- 不追求一次性完成全部模型和全部内容。


它应该同时保留三种参考的优点：

| 参考 | 吸收点 |
|---|---|
| 《小王子》 | 小星球、诗意、童话感、孤独而温柔的宇宙 |
| 《围攻》菜单 | 低多边形小星球、可旋转、世界选择感 |
| 多邻国路径 | 岛屿内部的清晰学习路径 |

但三者必须分层使用：

```txt
星系首页：小王子式星球宇宙
星球页：围攻式低多边形星球选择
岛屿页：多邻国式学习路径
概念页：严谨数学解释与局部关系图
图谱页：完整知识关系高级视图
```

这样既保留幻想感，又不会牺牲学习结构。
