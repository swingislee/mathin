## 2. 统一锚点（所有星球页面不可违背）

子页面可以有密度、可以有底色倾向、可以不留白，但以下锚点必须一致，这是"同一颗笔下的宇宙"的底线：

1. **纸感基底**：全站噪点纹理照常覆盖；星球页的底色只能是 `--paper` 之上的**低饱和罩色**（见各星球 `--p-wash`），不得整页换成高饱和背景。
2. **文字系统不变**：标题楷体 `font-display`、正文系统栈、字号阶梯同 01-4。
3. **线条语言不变**：手绘感描边 1.5–2px、圆点虚线星轨母题（每个星球页至少出现一次，形态可变：航线/灯串/等高线/藤蔓）。
4. **形状家族不变**：圆角家族（rounded-2xl 卡片 / full 按钮）、lucide 图标 strokeWidth 1.75。
5. **动效纪律不变**：01-5 的约束照旧；每个星球页最多允许一个专属轻动效（在各自小节中指明）。
6. **暗色 = 星夜**：各星球色板必须给出暗色值，气质随星夜统一变深变静。

实现机制：`SectionShell` 升级为接受 `planet` 属性，在最外层输出 `data-planet="earth|king|lamplighter|geographer|b612"`；`globals.css` 按 `[data-planet=…]` 作用域定义每星球的局部变量（命名统一为 `--p-accent / --p-accent-2 / --p-wash / --p-line`，含 `.dark` 变体）。组件只引用 `--p-*` 变量，禁止直接写星球色值。01-1.3 的板块 accent 表由本文件取代（story 的 accent 从 rose 改为狐狸橙）。

## 3. 各星球设计语言

每小节给出：色板（亮/暗）、装饰母题、布局个性、组件变体、专属动效、需要用户绘制的素材清单。**所有插画素材由用户提供或确认后才能上线（00-铁律 7）**；素材未就绪时用 `Star4`/几何占位。

### 3.1 故事 · 地球 `data-planet="earth"`

- **色板**：`--p-accent` 狐狸橙 `#DE9455`（暗 `#C97F45`）；`--p-accent-2` 夜航蓝 `#3B4668`（暗 `#8E9BC4`）；`--p-wash` 沙色罩 `#F9EFDC`（暗 `#1E2233`）；`--p-line` 沙棕 `#D9C3A0`（暗 `#3A4059`）。玫瑰红只作极小点缀（狐狸的玫瑰）。
- **装饰母题**：沙丘剪影水平分层线、旅行邮票/日戳、狐狸耳朵形小标记、星航虚线（星轨母题的"航线"形态）。
- **布局个性**：页面按"旅程"纵向推进——顶部是暖沙色地平线带（低饱和沙丘剪影 SVG，高度 ≤ 160px），章节时间线是一条穿过沙丘的斜向星航虚线，章节卡沿线交错左右分布。
- **组件变体**：章节卡 = 旅行明信片（4:3 封面 + 右上角"邮票"框放章节号 + 底部日戳式日期）；未发布章节 = 灰度 60% + 邮票框空置。
- **阅读页**：进入 `/story/[chapter]` 后切入"夜航"氛围——阅读区外圈用夜航蓝深色边纸（亮色模式也允许，这是内容承载页的例外），漫画居中，顶部细进度条用狐狸橙。
- **专属动效**：章节卡 hover 时邮票框内章节号轻微盖章下压（scale 0.97 一次性过渡）。
- **素材清单（用户绘制）**：沙丘地平线剪影 ×1、狐狸小像 ×1（空状态用）、邮票框线稿 ×1。

### 3.2 游戏 · 国王的星球 `data-planet="king"`

- **色板**：`--p-accent` 披风绛红 `#B0525C`（暗 `#C76D76`）；`--p-accent-2` 星徽金 `#D8B266`（暗 `#B99450`）；`--p-wash` 淡绛罩 `#F9EBE7`（暗 `#242030`）；`--p-line` `#E3C9BC`（暗 `#453A47`）。
- **装饰母题**：小王冠（难度符号：一至三顶王冠替代星级）、纹章盾形、卷轴上下沿线、"敕令"边框（细双线框）。
- **布局个性**：游戏大厅 = 谒见厅，卡片网格规整对称（`wide`），页首可有一行小字引用国王的话作题签（法文原文小注，同首页题签规格，不算宣传语）。
- **组件变体**：游戏卡 = 纹章盾形上徽记 + 名称 + 王冠难度 + "开始"按钮（该页 secondary 按钮描边用 `--p-accent-2` 金）；排行榜 = 御前名册——表格首列前三名用金/银/铜三色小星徽（`Star4` 填色），其余名次用墨色数字；个人最好成绩徽章挂卡片右上角如"御赐勋章"。
- **对局页**：计时器放"卷轴"横条内居中，庄重衬线数字（tabular-nums）；面板外框用敕令双线框。
- **专属动效**：完赛提交成功时名次徽章一次性轻微展开（scale 0.9→1，200ms）。
- **素材清单**：小王冠线稿 ×1、盾形纹章底 ×1、卷轴横条 ×1。

### 3.3 思维 · 点灯人的星球 `data-planet="lamplighter"`

- **色板**：`--p-accent` 灯焰黄 `#F1C86E`（暗 `#E7BC5F`）；`--p-accent-2` 暮色蓝紫 `#4A4A72`（暗 `#9C9CCB`）；`--p-wash` 暮霭罩 `#F3EFE9` 带极轻的紫灰倾向（暗 `#1C1E30`）；`--p-line` `#D5CFC9`（暗 `#383B54`）。
- **装饰母题**：路灯剪影、光晕同心圆（radial 渐变，极淡）、灯串虚线（星轨母题的"灯线"形态：虚线点略大、微发暖光）。
- **布局个性**：思维卡列表 = 一条街道的灯——卡片纵向单列（`max-w-3xl`），每卡左侧立一盏小路灯图标；页面顶部暮色渐变横带（高度 ≤ 120px，亮色模式下也允许偏暗，这是该星球的性格）。
- **组件变体**：思维卡 = 灯下卡片：左侧灯图标 + 思维点名称 + 一句白话解释；**未读灯灭**（灯头描边）、**读过灯亮**（灯头填 `--p-accent` + 极淡光晕）——已读状态首版存 localStorage，登录版后续迁库。
- **文章页**：正文窄栏居中，段落间用小灯点分隔符（替代通用星形分隔）。
- **专属动效**：卡片 hover 时灯头从描边到填充的 200ms 过渡（灯被点亮）——这是该页唯一动效，也是全站最点题的一处。
- **素材清单**：路灯线稿 ×1（灭/亮两态）、暮色横带 ×1（可用纯 CSS 渐变替代，可不绘）。

### 3.4 知识 · 地理学家的星球 `data-planet="geographer"`（核心板块）

- **色板**：`--p-accent` 保留王子绿 `--leaf`/`--leaf-deep`（生长中的知识）；`--p-accent-2` 墨蓝 `#3E4A5E`（暗 `#A7B4C9`）；`--p-wash` 羊皮纸罩 `#F8F1E2`（暗 `#20242F`）；`--p-line` 图纸格线 `#E2D8C2`（暗 `#363C4A`）。
- **装饰母题**：极淡的方格图纸底纹（仅此星球允许网格背景，格线用 `--p-line` 至多 6% 不透明度）、罗盘玫瑰小图、等高线弧、词条编号（每个概念一个"图鉴编号"，格式 `Nº 041`，衬线体）、"永恒之物"小印章（红色小方印，用 `--rose-deep`，全页至多一处）。
- **布局个性**：学习线总览 = 地图集——四个学段是四张"分区地图"（横向星轨在此变形为**等高线式航线**），概念是航线上的星点：未学 = 描边星点，已学 = 填充（进度 P2 落库，首版全描边）；每张分区图右上角小罗盘标注学段名。
- **概念页（词条）**：严格四段结构（02-3.3）不变，视觉上做成"图鉴词条"：标题行 = 编号 + 概念名 + 学段徽记；"定义"段首字下沉或左侧墨蓝竖线；"看见它"交互区外框用图纸格线框；"从哪来/到哪去"= 小型航线图（前置概念在左、后继在右，星点+虚线连接）；"试一试"= 三道题做成地理学家的"考察记录"清单样式。
- **专属动效**：概念星点 hover 时从描边到填充的过渡（与点灯人的灯呼应但无光晕）。
- **素材清单**：罗盘玫瑰小图 ×1、学段徽记 ×4（可先用文字替代）、"永恒之物"印章 ×1。

### 3.5 工具 · 商人的星球 `data-planet="businessman"`

- **色板**：`--p-accent` 账本金 `#C89B52`（暗 `#A87F42`）；`--p-accent-2` 墨灰 `#5C554A`（暗 `#B4AB9C`）；`--p-wash` 米金罩 `#F8F1DF`（暗 `#232019`）；`--p-line` 账线 `#E0D3B4`（暗 `#413A2C`）。
- **装饰母题**：账本横格线（细单线，行距均匀）、算珠（一排小圆点，其中一颗填金色）、星币小图标（星形＋圆框）、编号印花（工具编号 `Nº 07` 式，与地理学家词条编号同族但用金色）。
- **布局个性**：工具库 = 商人的工作台，`wide` 网格**严格对齐**（本星球的性格是精确）；分类 tab 做成一排算珠——当前分类的珠子填金色并移到"已拨"位置（位移 2px）；页面顶部可有一行小字法文题签（"Deux et cinq font sept…"），规格同首页题签。
- **组件变体**：工具卡 = 器具挂牌（缩略图 + 工具名 + 适用年级小徽 + 打开/嵌入两按钮），卡内元素基线严格对齐账本格线；"复制嵌入代码"成功反馈 = 星币图标翻面一次（150ms）。
- **全屏工具页**：顶部窄条左侧放当前工具挂牌小版 + 金色编号，画布区四角用极细角线框出（图纸感），其余全给工具；`/embed/[tool]` 纯净模式无任何装饰。
- **专属动效**：分类算珠的"拨动"位移（2px，200ms）——该页唯一动效。
- **素材清单**：算珠横排 ×1（可用纯 CSS 圆点替代，可不绘）、星币小图标 ×1。

## 4. 首页站点的星球化（P1 任务）

首页即 B-612：主视觉就是王子和他的星球，五个站点是他旅途中造访/抵达的五颗星球——入口与世界观由此闭环。站点改造：

- 每颗站点 = 24px 内的迷你星球 SVG：底色用该星球 `--p-accent` 的 30% 罩染 + 1.5px `--crater` 描边 + 一个特征细节（地球 = 一道沙丘弧线；国王星 = 顶部小王冠尖；点灯人星 = 一点灯焰黄；地理学家星 = 一圈经线弧；商人星 = 三粒金色算珠点）。
- 特征细节必须小到"凑近才认出"，远看仍是五颗安静的圆点星球——克制原则不变。
- 迷你星球 SVG 先由 agent 用纯几何线条起稿，截图给用户确认后才可合入；用户也可能直接手绘替换。
- hover 时站点下方标签从板块名切换为星球名（新增文案 key `nav.planetNames.*`，如「地理学家的星球」）；移动端胶囊不变。

## 5. 功能板块 = 「工作台」基调（不绑定星球）

dashboard / classroom / notebook / whiteboard 及登录注册是专业工作页面，**不使用星球主题**，走规整正式的工作台基调：

- 视觉只用基础设计系统（01）：`--paper/--ink/--line` + 01-1.3 表中的板块 accent；无绘本装饰母题（无邮票、王冠、灯、罗盘、算珠），星轨虚线也不出现。
- 保留的统一感来自底层：纸感噪点、楷体 h1、圆角家族、lucide 线宽、动效纪律——即"同一张纸上的正经页"。
- 信息密度向专业工具看齐：允许紧凑表格、多栏布局、密集表单；间距阶梯可降一档（区块 `space-y-10/6`）。
- 教室上课页、白板画布页以功能效率优先，装饰元素为零。

## 6. 给执行 agent 的补充铁律

1. 星球色值只在 `globals.css` 的 `[data-planet=…]` 作用域内出现一次，组件一律引用 `--p-*`。
2. 实现某板块页面前，先读本文件对应小节 + 02 对应小节；两者分别管"长什么样"和"有什么/怎么用"。
3. 每个星球页完成后按 00-铁律截图（亮/暗 × 桌面/移动）给用户审校，未确认不进入下一板块。
4. 素材清单里的插画一律等用户提供；先用几何占位并在 PR 描述中列出占位清单。
5. 各星球的"专属动效"每页只允许那一个，且必须尊重 `prefers-reduced-motion`。
