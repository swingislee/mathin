# Mathin 整体规划 · 02 页面规范

本文件规定首页与各板块页面的布局、交互与验收标准。所有实现必须使用 01 的 token 与组件。

## 1. 首页（`/[locale]`）重设计 —— 「月亮与五颗星球」

### 1.1 设计意图

现状问题：主图居中孤立、标题挤在 header 左上、五个导航链接是 header 里一排小字，没有层次也没有世界观。重设计把「小王子坐在月亮上」作为舞台，五个公开板块化为环绕月亮的五颗小星球，用星轨虚线相连 —— 导航本身就是画面的一部分。

### 1.2 桌面端布局（≥ lg）

```
┌────────────────────────────────────────────────────────┐
│ Mathin(小字logo)                [语言][主题][抽屉]      │  header：极简，不再放五个导航链接
│                                                        │
│   数学，从好奇开始              ✦                       │
│   （h1，文楷体，两行内）                                 │
│   在故事、游戏和动手探索中，                             │
│   发现数学清晰而有趣的一面。(muted)        ●story        │
│                                        ╱  (星轨虚线)   │
│   [开始探索 →]（唯一primary，        ● games            │
│     指向 /terms）                 ╱                     │
│                        ┌─ Main.png 主图 ─┐   ● minds    │
│              ✦         │  (约55%版面宽,   │  ╱           │
│                        │   右下偏移)      │ ● terms      │
│                        └─────────────────┘╱             │
│                                          ● tools        │
└────────────────────────────────────────────────────────┘
```

具体要求：

- 两栏 grid：左栏（约 40%）= 标题区，垂直居中；右栏（约 60%）= 主图 + 星球导航层。
- h1 用 `home.eyebrow` 文案「数学，从好奇开始」，字体霞鹜文楷 `text-4xl md:text-5xl`；下方 `home.description` 为 muted 正文；再下方一个 primary 按钮「开始探索」链接到 `/terms`（新增文案 key `home.cta`）。
- 主图 `Main.png` 放右栏，`max-w-[560px]`，允许底部被视口裁掉一点（月亮沉入页底的感觉），`priority` 加载。
- 五颗星球 `PlanetLink` 沿主图右上到右下呈弧线分布（绝对定位在右栏容器内，位置用百分比写死并在规范注释中标明），彼此以一条 `StarPath` 弧形星轨虚线相连。
- `PlanetLink` 规格：直径 56px 圆、`--card` 底、1.5px `--crater` 描边、内放板块 lucide 图标（story=BookOpen, games=Puzzle, minds=Lightbulb, terms=Sprout, tools=Wrench），圆下方 `text-sm` 板块名；hover：圆底变该板块 accent、`-translate-y-0.5`、标签变 ink 色。每颗星球带 `float` 漂浮动画，`animation-delay` 依次错开 1s。
- 页面不出现滚动条（`min-h-screen` 单屏完成）；`✦` 装饰星 3 颗，放在留白处。

### 1.3 移动端布局（< lg）

- 纵向：header → h1 + 描述 + CTA（居中）→ 主图（全宽，`max-w-[420px]`）→ 五个板块入口改为**两行网格**（3+2）的圆形入口，不做绝对定位弧线。
- 星轨虚线在移动端隐藏。

### 1.4 验收标准

- [ ] 1280×800 与 375×812 两种视口截图无布局破碎、无横向滚动。
- [ ] 五颗星球均可键盘 Tab 聚焦，焦点样式为 accent 描边环。
- [ ] 深色模式（星夜）下主图 `brightness-95`，星球与文字对比度 ≥ 4.5:1。
- [ ] `pnpm lint && pnpm typecheck && pnpm build` 通过。

## 2. 子页面统一骨架 `SectionShell`

所有板块页共用，替代现有 `SectionPage` 的"居中一张卡"布局：

```
SiteHeader（保留，含返回首页的 logo）
面包屑：首页 / 板块名（text-sm muted）
h1 板块名（文楷体）+ 标题左下 32px 短横线（--section-accent，2px 圆头）
可选一句板块简介（muted）
────────── 内容槽 children ──────────
页脚：一颗 ✦ + 「Mathin」小字（muted）
```

- Props：`{ section, accent, intro?, wide?, children }`；`wide` 时内容 `max-w-5xl`，否则 `max-w-3xl`。
- 受保护板块的「退出登录」按钮移入 UtilitySheet，不再放页面顶部。
- 未建设完成的板块内容槽渲染 `EmptyState`（一颗 star4 + `common.comingSoon`）。
- 现有 `src/app/[locale]/[section]/page.tsx` 的公开/受保护白名单机制保留。

## 3. 各板块页面规格

以下每小节 = 该板块首版（MVP）页面结构（信息架构与功能）。**五个公开板块的视觉语言（星球主题色板、装饰母题、布局个性、组件变体）以 `05-planet-themes.md` 对应小节为准**；两者配合使用：02 管"有什么、怎么用"，05 管"长什么样"。子页面不再受首页式留白约束，只需遵守 05-§2 的统一锚点。

### 3.1 story 故事

- `/story`：章节时间线。垂直 `StarPath` 星轨贯穿页面，每章一个节点卡：封面缩略图（4:3，占位插画）+ 章节号 + 标题 + 一句摘要 + 状态（已发布/敬请期待）。未发布章节卡为 60% 透明度且不可点。
- `/story/[chapter]`：漫画阅读页。首版 = 纵向长图滚动阅读（图片序列，`next/image`，懒加载），顶部进度条（rose 色），底部上一章/下一章。动画形式留待后续（预留 `media_type: comic | animation`）。
- 内容源：`content/story/<chapter-slug>/meta.json`（标题、摘要、顺序、状态）+ `pages/*.webp`。

### 3.2 games 游戏

- `/games`：游戏卡片网格（`wide`）。每卡：游戏图标、名称、一句规则简介、难度星级（1–3 颗 star4）、「开始」secondary 按钮；卡片右上角小徽章显示个人最好成绩（登录后）。
- `/games/[game]`：对局页。统一框架 = 顶部（计时器 + 难度选择 + 重开）+ 中部游戏面板（每个游戏实现 `GameBoard` 接口，见 03）+ 底部说明折叠区（教学功能：规则、技巧、相关知识点链接到 terms）。
- `/games/[game]/ranks`：排行榜。按难度 tab 切换，表格列 = 名次/昵称/用时/日期；未登录提示登录参与排名。
- 竞速计时在客户端计、服务端校验提交（见 03 反作弊约定）。首发游戏顺序：数独 → 数和（Kakuro）→ 幻方。

### 3.3 terms 知识（核心板块）

- `/terms`：学习线总览。按学段分组（小学/初中/高中/大学基础，四段），每段内是主题列表（如「数与运算」「几何」），主题下挂概念节点；视觉 = 每个学段一条水平星轨，概念是轨上的星点，已学/未学用填充/描边区分（学习进度 P2 再做，首版全部同态）。
- `/terms/[slug]`：概念页，固定四段结构：
  1. **定义**（MDX 正文，KaTeX 公式）；
  2. **看见它**（交互演示槽：注册表组件或复用 tools 的 embed）；
  3. **它从哪来 / 到哪去**（前置概念、后继概念的 `PlanetLink` 小圆链接，形成图谱导航）；
  4. **试一试**（1–3 道轻量自测题，首版纯前端判定）。
- 内容源：`content/terms/<slug>.mdx`，frontmatter：`title, stage, topic, order, deps[](前置slug), interactive?(组件id)`。图谱关系完全由 `deps` 推导，不单独维护关系表。
- 学习线的选题范围：小学到大学基础（微积分/线代入门为止），不含深入内容 —— 新增节点时如拿不准是否超纲，问用户。

### 3.4 minds 思维

- `/minds`：思维卡集合（`wide` 网格）。每卡 = 思维点名称（如「有序思考」「几何直观」）+ 一句白话解释；点开 `/minds/[slug]` = MDX 文章页（骨架同概念页但只有正文 + 「在哪些知识里出现」的反向链接列表，由 terms frontmatter 的 `minds[]` 字段反查）。
- 该板块定位未完全确定，首版只做卡片 + 文章的最小结构，不做更多交互，等用户想清楚再扩展。

### 3.5 tools 工具

- `/tools`：工具库（`wide` 网格），按分类 tab（数与运算/几何/行程问题/其他）。每卡：工具名、适用年级、缩略图、「打开」按钮 + 「复制嵌入代码」按钮。
- `/tools/[tool]`：全屏工具页。顶部窄条（工具名 + 返回 + 全屏按钮 + 复制嵌入代码），其余全部给工具画布。
- `/embed/[tool]`：**无 header/footer 的纯净嵌入路由**（供 iframe 插入课件与教室），同一工具组件复用；`?locale=zh|en` 控制语言。此路由在 proxy 的公开名单内。
- 工具用注册表模式（见 03），首批迁移用户已有 demo：分数数轴、相遇追击演示（迁移时逐个与用户确认交互细节）。

### 3.6 classroom 教室（复杂度最高，P4）

- `/classroom`：教师见「我的教室」列表 + 新建；学生见「我加入的教室」+ 邀请码加入。
- `/classroom/[id]`：教室主页 = 成员列表、课件资料、作业列表、历史课堂记录。
- `/classroom/[id]/live/[session]`：**上课页**。布局 = 主区课件画布（翻页由教师控制、学生端实时跟随）+ 右栏（成员/举手/答题）+ 底部工具条（插入 tools 工具、开白板、发题目）。教师可对学生答案实时评分批改，下课自动生成课堂报告（出勤、答题、得分）。
- 实时通道用 Supabase Realtime（见 03）。此板块必须拆成多个子任务执行，任何 agent 不得试图一次实现。

### 3.7 notebook 笔记

- `/notebook`：公开帖子流，支持最新/最热排序；未登录可浏览。
- `/notebook/[postId]`：公开帖子详情 + 点赞；点赞需登录。
- `/notebook/me`：个人 Notion 式笔记工作区，需登录。
- `/notebook/me/[noteId]`：在工作区内打开并编辑一篇笔记，需登录。

P3 已批准引入 BlockNote 富文本编辑器；完整范围、数据模型、保存策略、视觉与验收以 `07-p3-notebook.md` 为唯一执行依据。

### 3.8 whiteboard 画板

- `/whiteboard`：我的白板列表 + 新建 + 邀请协作（链接分享）。
- `/whiteboard/[id]`：画布页。首版能力：笔、橡皮、颜色（限设计系统 6 色）、粗细、撤销、清空、导出 PNG；多人协作光标与笔迹实时同步。技术路线见 03（首版 Canvas + Supabase Realtime 广播笔画；若延迟不可接受再升级 Yjs，需用户批准新依赖）。

### 3.9 dashboard 仪表盘

- `/dashboard`：三块卡片 —— 我的教室（近期课程）、我的游戏成绩（最好名次）、我的笔记（最近发布）。纯读取聚合页，最后做。

### 3.10 登录 / 注册页

- 现有逻辑保留，视觉统一：卡片居中 `max-w-sm`，卡片上方放一颗小星球插画位（首版用 `Star4`），标题文楷体，主按钮 primary。错误提示用卡内红字（rose），不用 alert。
