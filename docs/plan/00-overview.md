# Mathin 整体规划 · 00 总览与信息架构

> 本目录（docs/plan/）是 Mathin 网站的唯一权威规划。任何 agent 在实现功能前必须先读本文件与对应板块的规范；与规划冲突的实现一律不做，有疑问时停下来向用户确认，不要自行发挥。
>
> **与 P4I 的关系（2026-07-22 追记）**：本文写作时间早于 `19-p4i-final.md`（P4I，2026-07-20 立项），学校端后台（教师/教务/教研/学辅等角色）相关表述如与 doc 19 冲突，以 doc 19 为准；本文对访客/学生/家长等公开板块和整体产品定位的描述不受影响。

## 1. 产品定位

Mathin 是一个中英双语的数学探索网站，以《小王子》的视觉世界观贯穿全站。它同时服务三类人：

| 角色 | 使用方式 |
| --- | --- |
| 访客 / 学生（未登录） | 浏览故事、玩益智游戏、自学知识线、使用公开工具 |
| 学生（登录） | 参加竞速排名、进教室上课、发笔记、用画板 |
| 教师（登录，role=teacher） | 调用教学工具、开设教室、批改评分、管理内容 |

核心初衷（最高优先级）：**「知识（Terms）」板块** —— 把小学到大学基础数学的定义与知识点串成一条可自学的、图形化交互的学习线，替代书本式教学。其他板块围绕它展开。

## 2. 九大板块定位

| 板块 | 路由 | 访问权限 | 一句话定位 |
| --- | --- | --- | --- |
| 故事 story | `/story` | 公开 | 用漫画/动画形式的完整故事线串联数学发展史 |
| 游戏 games | `/games` | 公开（排名需登录） | 数独、数和、幻方等 puzzle，竞速排名 + 教学功能 |
| 思维 minds | `/minds` | 公开 | 揭示数学学习背后的思维方式（有序思考、几何直观等） |
| 知识 terms | `/terms` | 公开 | 核心板块：小学→大学基础数学的定义学习线，图形化交互呈现 |
| 工具 tools | `/tools` | 公开 | 课堂交互工具（分数数轴、行程问题演示等），可嵌入课件 |
| 教室 classroom | `/classroom` | 登录 | 在线教学平台：多人同步课件、实时评分批改、课堂报告 |
| 笔记 notebook | `/notebook` | 登录（浏览可放开） | 社交板块：所有人发布零散知识文章，互相点赞 |
| 画板 whiteboard | `/whiteboard` | 登录 | 协作白板，独立使用 + 嵌入在线课堂 |
| 仪表盘 dashboard | `/dashboard` | 登录 | 个人入口：我的课程、成绩、笔记、待办 |

導航层级：首页只展示 5 个公开板块（story/games/minds/terms/tools）；4 个功能板块（dashboard/classroom/notebook/whiteboard）收在右上角的实用功能抽屉（UtilitySheet）里，登录后可见。此结构已实现，保持不变。

## 3. 权限模型

- `profiles` 表存 `role`：`student`（默认）| `teacher` | `admin` | `parent`（P4B 起）。注册即 student；其他角色由 admin 授予（P4B 起走后台员工管理页，见 `10-school-backend.md`）。
- 页面访问控制沿用现有两层：`src/proxy.ts` 乐观跳转 + 页面内 `requireUser()`；数据授权靠 Supabase RLS（见 03-data-and-tech.md）。
- 教师专属能力（开教室、批改、管理内容）在服务端校验 `profiles.role`，不信任前端。

## 4. 内容策略（重要约定）

- **课程性内容（terms 知识线、minds 思维卡、story 章节脚本）以 MDX 文件形式放在仓库 `content/` 目录**，随 git 版本管理，不入数据库。理由：作者即开发者、需要 review、结构化引用。
- **用户产生的内容（笔记、点赞、成绩、课堂记录、白板数据）入 Supabase**。
- 游戏和工具是代码，不是内容：各自用「注册表模式」在代码里登记（见 03）。

## 5. 给执行 agent 的铁律

1. 先读 `AGENTS.md`，再读本目录对应文档，再动手。
2. 未在规划中列出的依赖不得新增；需要新库时列出候选与理由，等用户批准。
3. 所有颜色、字体、间距必须使用 `01-design-system.md` 定义的 token，禁止硬编码色值。
4. 所有文案必须走 next-intl（`messages/zh.json` + `messages/en.json` 同步添加），禁止硬编码中文/英文。
5. 每个任务按 `04-roadmap.md` 的阶段与验收标准执行，不跨阶段"顺手"实现未排期功能。
6. 完成后必须通过：`pnpm lint`、`pnpm typecheck`、`pnpm build`。
7. 视觉还原以 `public/Main.png` 与设计系统为准；不确定的视觉决策（新插画、新配色）停下来问用户。
