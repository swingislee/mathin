## 生命周期覆盖

导入线索 → 活动报名 → 到场升档 → 持续跟进 → 报名缴费 → 课堂点名 → 多维课评与知识总结 → 视频上传/倍速审阅/家长回看 → ≤3 课次续费提醒 → 流失池回流留痕，均有对应页面、Server Action/RPC、RLS 或白名单 RPC。

## 自动门禁

- `pnpm p4d:audit`：P4D 关键静态契约。
- `pnpm lint`、`pnpm typecheck`、`pnpm build`。
- `scripts/verify-p4d-db.sql`：事务内以固定五角色执行活动容量、到场升档、跟进追加、教师整班课评、学生/家长课评读取；末尾 `ROLLBACK`，不污染测试数据。开发库已通过。
- 真实 Storage/API：固定学生账号上传私有视频并写入元数据，固定教师账号审阅，学生本人可回读，家长不能直接读取 `session_videos`、但可由 `get_my_reviewed_videos()` 读取已审结果；1 秒 signed URL 在有效期内返回 200、过期后返回 400；临时对象与元数据均已清理。开发库已通过。
- 部署元数据核验：bucket 私有且上限 200 MiB；课程/讲次物理删除和视频 `deleted_at` 直改均已撤权；视频删除、上传校验、本人读取白名单函数均可由 `authenticated` 执行。

## 审计发现与修复

真实 REST 回归发现学生视频元数据插入及即时回读的策略子查询会被 `students` / `enrollments` 自身 RLS 隐式过滤。已以追加迁移 `20260712000900`、`20260712001000` 引入最小范围的 `security definer` 判断函数，并重新跑通端到端链路。
